Office Supplies Inventory

Important Notes: Due to company privacy and confidentiality policies, I am unable to share or publish the actual projects I developed at work. Instead, this portfolio contains descriptive pages that outline the type of systems I worked on without exposing proprietary code or sensitive data.

Login Page

  • Office Supplies Login Page — Employees can log in without registration. The default password is their EmployeeCode, which they may optionally change using change password function of the system.

Landing Page

  • The landing page displays all items available for request. On the left side, a category list allows users to filter items by type. Beside the search bar, a row of icons represents different processes: Item Surrender, Cart, Transactions/Records, Confirmation Access, Material Consumption Report, Price Change History, and Resigned Staff (with ongoing requests). On the right, next to the user avatar, are Notifications (including Request Notifications, Stock Request Notifications, and Surrendered Notifications). The clock icon sets the cutoff date and time, mirroring the schedule shown at the top center of the page. The clipboard icon provides access to staff assignment, confirmation, delivery, and issuance functions.

Request Item Dialog

  • Request Item Dialog This dialog is displayed when clicking the Cart icon at the bottom right of each item. It allows the user to input the quantity and purpose of the request. The user can choose Add to Cart to continue selecting additional items, or select Send Request to immediately notify the confirmation in-charge about the request.

Surrendered Items Dialog

  • Surrendered Items Dialog This dialog displays all surrendered items in the main table, with status summaries shown at the top of the page. Users can filter and search records for easier tracking. By clicking Surrender Item, the selected items are added to the surrender list. The table on the right provides additional information such as status, surrendered by, and item description.

Cart

  • Cart Function The cart displays all items selected for request after setting their quantities. Users can review each item, adjust quantities if needed, and specify the purpose of the request. From here, they may either continue adding more items or proceed to submit the request for confirmation.

Transactions/Records

  • Transactions/Records This page displays all requested items along with their details and current status. Each record includes reference number, item information, quantity, total price, date requested, and status updates. Users can filter or search transactions for easier tracking. A rating feature is also included to evaluate the inventory process, providing feedback on efficiency and accuracy.

Rating

  • Rating Function This feature allows users to evaluate the inventory process based on defined criteria such as quality, delivery, communication, response to urgent concerns, and problem resolution. Ratings are recorded using percentage ranges, with corresponding performance levels: 60%–79%: Needs Improvement 80%–89%: Satisfactory 90%–99%: Very Good 100%: Excellent Users can also provide remarks for each criterion, ensuring that feedback is both quantitative and descriptive. This helps track service performance and supports continuous improvement in the inventory workflow.

Request Confirmation Access

  • Confirmation Access This module manages the processing of approved requests by the inventory team. Once a request has been approved by the respective department or team, it is forwarded here for confirmation and issuance. The interface provides visibility into all received requests, their approval dates, requested items, quantities, purposes, and current status. Assigned staff can track progress, notify requesters of pending pickups, and oversee the completion of issuance and storage activities.

Stock Replenishment Tab

  • Stock Replenishment Tab This section handles inventory requests for replenishing items. Once a replenishment request is submitted, it is processed and tracked here. The Billing team is in charge of managing this module, ensuring that requests are properly endorsed, approved, and delivered. The interface provides detailed records including request codes, dates, requester information, items, quantities, purposes, and current status, allowing for transparent monitoring of the replenishment workflow.

Stock Monitoring

  • Stock Monitoring Tab This section provides a comprehensive view of inventory levels and item statuses. It tracks selected items for replenishment, showing details such as item codes, descriptions, unit of measure, stock quantity, total issued, confirmed deliveries, received secondhand items, and undelivered quantities. Items flagged with a Re-Order status indicate the need for replenishment, ensuring that the inventory team can proactively manage supply levels and avoid shortages.

Stock Monitoring Details

  • This section displays the detailed stock history of a specific item, organized by cut-off periods. Each record shows the previous stock, delivered quantity, received secondhand quantity, issued quantity, and resulting stock within the defined cut-off date range. The system allows filtering by year and month, as well as setting a cut-off start day, ensuring accurate monitoring of stock movement over time. This feature provides transparency in inventory flow and helps identify when replenishment is required.

Item Request Details

  • Per Item Records This section consolidates all records related to a specific item, providing a detailed view for accurate inventory checking. It tracks issued, delivered, and received secondhand materials within defined cut-off periods, ensuring transparency in item movement. Each record includes reference numbers, requester details, item descriptions, quantities, and dates, allowing the inventory team to verify requests and monitor stock flow more effectively. This feature supports precise auditing and strengthens overall inventory control.

Cost Record

  • This module provides a detailed breakdown of manpower and cost consumption across departments, teams, and individual employees. Users can filter records by team or employee, allowing precise monitoring of resource allocation and expenses. The dashboard summarizes overall departmental costs while also showing itemized consumption per staff member, ensuring transparency and accountability in inventory usage and financial tracking.

Consumption Report

  • Budget tracking — seeing which months drive the highest expenses.
  • Inventory planning — identifying when to replenish stock before shortages occur.
  • Department analysis — comparing usage across teams for accountability.
  • Forecasting needs — predicting future demand based on past consumption patterns.
