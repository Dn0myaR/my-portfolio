Login Page

  • GatePass login interface with employee authentication, password visibility toggle, and integrated access request.

Access Request

  • access requests routed for approval—no registration required.

Landing Page

  • The landing page displays all incoming and outgoing materials in a table, with their request type and status, sorted by descending GatePass number.

Request Form

  • Request Form dialog that captures requester details, request type, and delivery/pick‑up location, alongside a table of listed materials. The dialog integrates key functions: Print Preview for PDF output, Status to display the full request process, Unforward (visible only when status is Forward), file upload for signed GatePass records, and dynamic creation of new gatepasses— either Pull‑out or Delivery depending on the existing request type.

PDF

  • This is the output of Print Preview, with the design based on the sister company’s request requirements.

Status Card

  • The Status button displays the complete history of a GatePass request, showing all status records with timestamps and the names of individuals involved at each stage of processing.

Attachmen Display

  • The Attachment button is displayed only when a request includes an attached file, allowing users to view or manage uploaded GatePass documents.

Create Request

  • first way, New GatePass creation can be done by loading materials directly from the Request Management System (RMS). By inputting the RMS Code, all material details are automatically retrieved—providing a shortcut when the materials already have an associated request in RMS.

Manual Create Request

  • The second method of GatePass creation is manual entry, where all details and materials are input using the Add Item button. For items not found in the master table, the Encode Item function is used. In contrast, the Load RMS option represents the first method, which automatically retrieves material details from the Request Management System (RMS).
